# Roadmap

## Phase 1 — Public foundation

- Create public repository base
- Publish initial README, status, and project scope
- Add license and basic project structure
- Reconstruct core documentation

## Phase 2 — State model

- Define peer states
- Define transition semantics
- Define interruption / return model
- Define controlled re-entry logic
- Define replay-aware communication assumptions

## Phase 3 — Reference implementation

- Build first protocol baseline
- Implement explicit state handling
- Implement reconnection and resynchronization logic
- Add instrumentation hooks
- Add reproducible local test cases

## Phase 4 — Multi-node testing

- Prepare heterogeneous test nodes
- Test under unstable or intermittent network conditions
- Observe synchronization quality
- Observe reconnection behavior
- Observe state consistency and overhead

## Phase 5 — Documentation and publication

- Publish protocol notes
- Publish implementation notes
- Publish test methodology
- Publish field evaluation results
- Improve repository structure for public reuse
